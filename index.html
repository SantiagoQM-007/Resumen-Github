Guía Interactiva de Comandos Git

Explicaciones detalladas y ejemplos de uso

Configuración Inicial

Antes de comenzar a usar Git, es necesario configurar tu identidad. Estos comandos establecen tu nombre y correo electrónico, que se usarán para identificar tus contribuciones en cada commit.

git config --global user.name "Tu Nombre"
git config --global user.email "tu@email.com"
Ejemplo de uso:
git config --global user.name "María García"
git config --global user.email "maria.garcia@ejemplo.com"
Nota: El flag --global hace que esta configuración se aplique a todos tus repositorios. Si quieres una configuración diferente para un repositorio específico, omite este flag y ejecuta los comandos dentro del repositorio.

Comandos Básicos

git init

Inicializa un nuevo repositorio Git en el directorio actual, creando un subdirectorio oculto .git que contiene toda la base de datos y la estructura necesaria para el control de versiones.

git init
Ejemplo:
mkdir mi-proyecto
cd mi-proyecto
git init
# Salida: Initialized empty Git repository in /ruta/a/mi-proyecto/.git/

git clone

Crea una copia local de un repositorio remoto. Esto descarga todas las versiones de cada archivo en el proyecto.

git clone [url]
Ejemplo:
git clone https://github.com/usuario/repositorio.git
cd repositorio

git add

Añade archivos al área de preparación (staging area). Los cambios marcados serán incluidos en el próximo commit.

git add [archivo]
git add . # Añade todos los archivos modificados
Ejemplo:
git add index.html # Añade solo un archivo
git add css/ # Añade todo el directorio css
git add . # Añade todos los archivos modificados

git commit

Registra los cambios en el repositorio con un mensaje descriptivo que explica qué cambios se han realizado.

git commit -m "mensaje descriptivo"
Ejemplo:
git commit -m "Añadir funcionalidad de inicio de sesión"

git status

Muestra el estado actual del repositorio: archivos modificados, añadidos al área de preparación o no rastreados.

git status

git log

Muestra el historial de commits, incluyendo autor, fecha y mensaje de cada commit.

git log
git log --oneline # Versión resumida
Ejemplo:
git log --oneline -n 5 # Muestra los últimos 5 commits en formato resumido

Ramas (Branches)

Las ramas en Git te permiten desarrollar funcionalidades aisladas unas de otras. La rama predeterminada es main (anteriormente master).

git branch

Lista todas las ramas locales en el repositorio actual. La rama activa se marca con un asterisco.

git branch
Ejemplo:
git branch
* main
feature-login
bugfix-header

git branch [nombre]

Crea una nueva rama, pero no cambia a ella automáticamente.

git branch [nombre-de-rama]
Ejemplo:
git branch feature-newsletter

git checkout

Cambia entre ramas o restaura archivos del árbol de trabajo.

git checkout [nombre-de-rama]
Ejemplo:
git checkout feature-newsletter
# Salida: Switched to branch 'feature-newsletter'

git checkout -b

Crea una nueva rama y cambia a ella inmediatamente.

git checkout -b [nombre-de-rama]
Ejemplo:
git checkout -b feature-pagos
# Salida: Switched to a new branch 'feature-pagos'

git merge

Combina los cambios de una rama específica con la rama actual.

git merge [rama]
Ejemplo:
git checkout main
git merge feature-login
# Esto fusiona la rama 'feature-login' dentro de 'main'

git branch -d

Elimina una rama después de que sus cambios hayan sido fusionados.

git branch -d [rama]
Ejemplo:
git branch -d feature-login
# Salida: Deleted branch feature-login (was 3e8f21a).
Nota: Usa git branch -D [rama] para forzar la eliminación de una rama que tiene cambios no fusionados.

Trabajo Remoto

Los comandos para trabajar con repositorios remotos te permiten colaborar con otros desarrolladores, compartiendo cambios a través de un servidor centralizado como GitHub, GitLab o Bitbucket.

git remote add

Vincula tu repositorio local con un repositorio remoto. Normalmente se usa "origin" como nombre del repositorio remoto principal.

git remote add [nombre] [url]
Ejemplo:
git remote add origin https://github.com/usuario/repositorio.git

git push

Envía los commits de tu rama local a la rama correspondiente en el repositorio remoto.

git push [remoto] [rama]
Ejemplo:
git push origin main

# Para configurar el seguimiento y usar simplemente 'git push' después
git push -u origin feature-auth

git pull

Obtiene los cambios del repositorio remoto y los fusiona automáticamente con tu rama local actual. Es una combinación de git fetch + git merge.

git pull [remoto] [rama]
Ejemplo:
git pull origin main

git fetch

Descarga cambios del repositorio remoto, pero no los fusiona con tu rama local. Esto te permite revisar los cambios antes de integrarlos.

git fetch [remoto]
Ejemplo:
git fetch origin
git diff origin/main # Ver diferencias antes de fusionar
Diferencia entre pull y fetch: git pull descarga y fusiona automáticamente, mientras que git fetch solo descarga, permitiéndote revisar los cambios antes de fusionarlos manualmente con git merge.

Otros Comandos Útiles

git diff

Muestra las diferencias entre archivos, commits o ramas.

git diff # Diferencias entre área de trabajo y área de preparación
git diff --staged # Diferencias entre área de preparación y último commit
git diff [commit1] [commit2] # Diferencias entre dos commits
Ejemplo:
git diff main..feature-auth # Diferencias entre dos ramas

git reset

Deshace cambios o commits dependiendo de las opciones utilizadas.

git reset [archivo] # Deshacer git add para un archivo específico
git reset # Deshacer git add para todos los archivos
git reset --soft HEAD~1 # Deshacer último commit manteniendo los cambios
git reset --hard HEAD~1 # Eliminar último commit y sus cambios
Ejemplo:
git add .
git reset index.html # Saca index.html del área de preparación
Precaución: git reset --hard elimina permanentemente los cambios. Úsalo con cuidado.

git stash

Guarda temporalmente los cambios que no están listos para ser committeados, permitiéndote cambiar de rama sin perder tu trabajo en progreso.

git stash # Guarda cambios en el stash
git stash list # Lista todos los stashes
git stash pop # Aplica y elimina el último stash
git stash apply # Aplica el último stash sin eliminarlo
Ejemplo:
git stash save "Cambios en formulario de login"
git checkout main
git checkout feature-auth
git stash pop

git tag

Marca puntos específicos en la historia del repositorio, normalmente usados para marcar versiones o releases.

git tag # Lista todas las etiquetas
git tag [nombre] # Crea una etiqueta ligera
git tag -a [nombre] -m "mensaje" # Crea una etiqueta anotada
Ejemplo:
git tag -a v1.0.0 -m "Primera versión estable"
git push origin v1.0.0 # Sube la etiqueta al remoto

Flujo de Trabajo Común

Un flujo de trabajo típico para colaborar en proyectos con Git incluye estos pasos:

  1. Clonar el repositorio o iniciar uno nuevo
    git clone https://github.com/usuario/repositorio.git # o git init
  2. Crear una nueva rama para trabajar
    git checkout -b feature/nueva-funcionalidad
  3. Realizar cambios en los archivos
    # Editar archivos con tu editor preferido
  4. Revisar los cambios
    git status
    git diff
  5. Añadir los cambios al área de preparación
    git add .
  6. Hacer commit de los cambios
    git commit -m "Añadida funcionalidad X para resolver Y"
  7. Actualizar tu rama con los cambios del remoto
    git pull origin main
  8. Subir los cambios al repositorio remoto
    git push origin feature/nueva-funcionalidad
  9. Crear Pull Request en GitHub/GitLab/Bitbucket
    Esto se hace a través de la interfaz web del servicio de alojamiento de Git.
  10. Después de la revisión, fusionar la rama a main
    git checkout main
    git merge feature/nueva-funcionalidad
    git push origin main
Buenas prácticas:
  • Haz commits pequeños y frecuentes con mensajes descriptivos
  • Actualiza tu rama desde la rama principal regularmente
  • Resuelve conflictos lo antes posible
  • Usa convenciones de nombrado para ramas (feature/, bugfix/, hotfix/, etc.)
  • Implementa revisiones de código a través de pull requests
